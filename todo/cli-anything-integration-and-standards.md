## 背景

在 agent CLI 的开发中，部分能力可以复用 `CLI-Anything`，避免重复建设；同时需要把其中成熟规则抽取为本项目可执行标准。

## 目标

### A) 已有 CLI 能力复用（避免重复开发）

- 若 `CLI-Anything` 已有可用组件/CLI，优先采用“引用机制”而不是重复实现。
- 定义可追踪的引用策略（版本、来源、升级方式、兼容约束）。

### B) 复杂开源项目 CLI 化流程

- 对复杂且有源码的目标项目，建立“基于 CLI-Anything 的快速 CLI 化”完整流程。
- 流程需覆盖：初始化开发、测试验证、发布、后续升级与回归验证。

### C) 规则选择性继承并机制化落地

- 从 `CLI-Anything` 中选择性抽取规范（而非全盘照搬），重点包括：
  - 文档格式规范
  - 命令行命名与参数规范
  - 输出与错误处理规范（含 JSON 模式）
- 将规范固化为本项目机制，并尽量在 commit 前自动执行基础校验。

## 计划任务

1. 梳理 `CLI-Anything` 可直接复用的 CLI/组件清单。
2. 设计本项目“引用机制”（目录布局、元数据、版本锁定与升级入口）。
3. 设计并文档化“复杂项目 CLI 化”标准作业流程（SOP）。
4. 提炼并发布本项目 CLI 规范文档（从 CLI-Anything 规则选择性继承）。
5. 增加 commit 前自动校验（如格式检查、命令规范检查、文档结构检查）。
6. 选取 `proxmox-cli` 或后续首个应用作为试点验证整套机制。

## 验收标准

- 已存在的 CLI-Anything 能力可通过统一机制引用，不再重复开发。
- 复杂项目 CLI 化流程可执行，且有升级回归路径。
- CLI 规范形成文档与自动化检查，commit 前可执行基础校验。
